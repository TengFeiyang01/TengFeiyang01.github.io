A. New Palindrome

题目大意

给你一个仅包含小写字母的回文字符串 sss ，问你能否将其变为另一个回文串。

思路

统计出现次数为奇数的字符的个数 oddoddodd ，再额外统计一下出现次数为一次的字符的个数 oneoneone 已经总的字符个数 cntcntcnt , 如果 cnt=1cnt = 1cnt=1 （仅有一个字符），或者 one = 1 \and cnt = 2 。直接输出 NONONO 。判断时可以直接比较 sss 的前半段是否是单个字符重复即可。

C. Contrast Value

题目大意

给你一个数组 aaa ，定义一个数组的 contrastcontrastcontrast 为相邻两项的差的绝对值之和。问你从 aaa 中去掉一些数后的数组 bbb 的最大长度是多少。

bbb 的 contrastcontrastcontrast 需要和 aaa 相等，且 bbb 最少有一个元素。

思路

考虑这样一个序列：1,3,5,9{1, 3, 5, 9}1,3,5,9 ，\abs{3 - 1} + \abs{5 - 3} + \abs{9 - 5} = 8 = \abs{9-1} ，所以对于连续上升的一段序列，我们仅需保留其头和尾即可，对于下降同理。所以我们可以统计 aaa 数组的峰和谷的个数 cntcntcnt ，那么答案就是 cnt+2cnt + 2cnt+2 。

需要特别注意的是，当总的和为 000 时，仅需保留一个数即可。

D. Red-Blue Operations

题目大意

给你一个数组 aaa 和 qqq 个独立的询问，每个询问给你一个整数 kkk ，初始时， aaa 中所有元素均为红色。

你必须恰好操作 kkk 次，第 iii 次操作 （ iii 从 111 开始）可以：

  • 把一个红色的元素加 iii ，同时变为蓝色。
  • 把一个蓝色的元素减 iii ，同时变为红色。

目标是最大化 aaa 的最小值。

思路

显然从前往后依次加 k,k−1...1k,k-1...1k,k−1...1 是最优的。

我们可以预处理出来 aaa 数组从前往后加了 kkk 个元素之后的最小值，记为 f[k]f[k]f[k] ，考虑此时操作的个数。

当 k≤nk\le nk≤n 时，数组最小值为 min(f[k],a[k])min(f[k], a[k])min(f[k],a[k]) ，（前 kkk 个位置加了之后可能导致第 k+1k+1k+1 个位置成为最小值）。

当 k>nk>nk>n 时，为了使得减少的次数最小，我们可以两两配对，每次 +i,−(i+1)+i, -(i + 1)+i,−(i+1) ，这样可以使得某个数减少 111 。

​ 如果此时加到 nnn 之后剩余的个数是偶数，我们总共可以凑出来 (k−n)/2(k - n) / 2(k−n)/2 个 −1-1−1 ，此时数组的最小值 mn=f[n]+k−nmn = f[n] + k - nmn=f[n]+k−n 。

​ 如果剩余操作的个数是奇数，那么我们需要把加到 ana_{n}an​ 处的元素拿出来，来抵消多余那一次减法，此时总共能配对 (k−n+1)/2(k - n + 1) / 2(k−n+1)/2 个 −1-1−1 ，此时数组的最小值 mn=min(f[n−1]+k−(n−1),a[n−1])mn = min(f[n - 1] + k - (n - 1), a[n - 1])mn=min(f[n−1]+k−(n−1),a[n−1])

我们将这 ttt 个 −1-1−1 每次贪心加在最大的位置，然后计算将所有元素减为最小值的操作次数 sss

​ 如果 s≥ts\ge ts≥t ，那么不会影响最小值 mnmnmn ，否则需要将多出来的 t−st-st−s 个 −1-1−1 均匀的分配到 aaa 数组中，此时答案为 $mn - \lceil \frac{(t-s)}{n}\rceil $ ，此时这个式子是等于数组的平均值下取整的。
